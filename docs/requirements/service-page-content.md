## Section 1: ヒーロー

### レイアウト
- **全幅ダーク背景**（`#0a0015` 〜 `#000000`）、`min-h-[80vh]`
- 背景: 抽象的な 3D / パーティクル / ループ動画（不透明度を抑え「環境」として配置）
- テキストは画面左寄せ or 中央

### コンテンツ

```
[装飾] "SERVICE" ── 極端に巨大化（10vw+）、opacity: 0.08 程度で背景グラフィックとして配置

[メインタイトル]
テクノロジーで、
事業の未来を設計する。

[サブタイトル / 英語]
Designing the future of business with technology.

[リード文]（小さく、上品に）
STARUPは3つのプロダクトを通じて、
企業のAI活用とデータドリブンな意思決定を支援します。
```

> **NOTE**: テキストは仮。あとから差し替え前提。  
> 装飾的な巨大英字は読ませるためではなく、画面のグラフィック要素として機能させる。

### アニメーション
- テキストのスタガード表示（GSAP）
- 背景素材はゆっくりとしたループ / 微細な動き

## Section 2: リード（トランジション）

### レイアウト
- 白背景
- 広大な余白（上下 `py-[15vh]` 以上）
- テキスト幅は狭く（`max-w-[600px]`）、左寄せ

### コンテンツ

```
[装飾番号] "01 — 03"（モノスペース、薄いグレー、大きめ）

[コピー]
深い技術力を核に、
3つの領域でAIの価値を届ける。

[補足]（さらに小さく）
SaaS開発から業界特化ソリューションまで、
一貫した技術基盤で支えるサービス群。
```

> **NOTE**: このセクションは「間」を作る役割。情報量は最小限でいい。

## Section 3: サービス一覧（メインコンテンツ）

### レイアウト方針
- **ブロークングリッド**: 3 枚のカードを均等に並べず、不規則に配置
- 各カードは**異なるサイズ・位置**で配置し、リズムを作る
- カード間にオーバーラップや余白の緩急をつける

### カード構成（3 サービス）

### カード共通デザイン

| 要素 | スタイル |
|---|---|
| ラベル番号 `01` | モノスペース、`text-7xl` 以上、`opacity-10`〜`opacity-20`、カードの装飾として配置 |
| カテゴリ | `text-xs`、`tracking-widest`、`uppercase`、グレー |
| タイトル | `text-3xl`〜`text-5xl`、`font-bold`、黒 |
| キャッチ | `text-lg`、通常ウェイト |
| 概要 | `text-sm`〜`text-base`、`text-gray-600`、`max-w-[400px]` |
| 画像 | `rounded-lg`、ダークオーバーレイ or モノクロフィルタで「環境」化 |
| CTA | テキストリンク + 矢印、ホバーでスライドアニメーション |

### アニメーション
- スクロール連動でカードがスタガード表示（GSAP ScrollTrigger）
- 画像はスクロールに応じて微妙にパララックス
- ラベル番号は先に薄く表示 → カード本体がフェードイン

## データソース

既存の `src/data/service/service.ts` は**トップページの ServiceSection 用**のため、  
Service 一覧ページ用に新しいデータファイルを作成する。

```
src/data/service/servicePageData.ts（新規）
```

### データ構造（案）

```typescript
export interface ServiceItem {
  id: string            // "starup-ai" | "archaive" | "send-ai"
  number: string        // "01" | "02" | "03"
  category: string      // "AI Platform" | "Data Platform" | "Supply Chain AI"
  title: string         // "STAR UP.AI"
  catchphrase: string   // キャッチコピー
  description: string   // 概要（1〜2文）
  image: string         // 画像パス（あとから差し替え）
  href: string          // 詳細ページへのリンク
  externalLink?: string // 外部サイトリンク（任意）
}
```
