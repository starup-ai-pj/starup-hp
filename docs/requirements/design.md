# Service ページ デザインガイドライン

## 設計思想

既存サイトの「ダーク × テック × ミニマル」のトーンを踏襲しつつ、  
以下4つの原則で**デザイナーが作ったと感じるクオリティ**へ引き上げる。

---

## 1. アセットの質 ── 素材を「環境」として扱う

### 方針
- フリー素材をそのまま貼らない。使う場合は**不透明度を下げる / モノクロ化 / ブラー**をかけ、「テクスチャ」として背景に溶かす。
- 既存サイトの Three.js（ポータルコア）の延長として、**Spline 3D / Lottie / アブストラクトなループ動画**をヒーロー背景に採用可能。
- AI 画像生成を使う場合は「具体的な人・風景」ではなく、**ミニマルな 3D ワイヤーフレーム / 抽象テクスチャ**の素材メーカーとして割り切る。

### 既存サイトとの接続
| 既存要素 | Service ページでの展開 |
|---|---|
| Three.js ポータル（ServiceSection） | ヒーロー背景に軽量な 3D / パーティクル演出を配置。ただし主役にしない |
| グリッチエフェクト（globals.css） | アクセント程度に限定使用。乱用しない |
| カラー `#9b59b6` `#3498db` `#e74c3c` | 抽象背景のアクセント光源色として踏襲 |

---

## 2. エディトリアル（雑誌風）レイアウト

### タイポグラフィ
- **ジャンプ率を極端に取る**：見出し英字は `clamp(4rem, 10vw, 12rem)` レベルまで大胆に巨大化。本文は `1rem` 前後で上品に。
- 象徴的な数字・英語（`"SERVICE"` `"01"` など）を**装飾グラフィック**として扱う。読ませるテキストとは別レイヤー。
- フォント: 既存の Alliance（本文）を継続。英字装飾には **Inter / モノスペース系** でコントラストを出す。

### 余白（マクロホワイトスペース）
- セクション間は `min-h-[30vh]` 以上の空白を恐れない。
- 要素の周囲に「何もない広大な空間」を意図的に作り、**余白自体をデザインの主役**にする。
- コンテンツ幅は既存 `max-w-[1500px]` を踏襲しつつ、テキストブロックは `max-w-[700px]` 程度に絞る。

### 情報の引き算
- 全てを言葉で説明しない。サービスカードには**キャッチ + 1〜2文の概要**のみ。
- 詳細は各サービス詳細ページに委ね、一覧ページは「雰囲気で惹きつける」ことに集中する。

---

## 3. 自由な座標と重なり ── ブロークングリッド

### オーバーラップ（交差）
- 画像の上にテキストを半分被せる、親枠から要素をはみ出させるなど、**奥行き（レイヤー）** を持たせる。
- 実装: `relative` + `negative margin` / `translate` / `absolute` 配置を組み合わせる。

### 非対称レイアウト
- サービスカード 3 枚を均等グリッドに並べるのではなく、**大きさ・配置をわざと不規則に散らす**。
- 例: 1枚目を大きく左寄せ、2枚目を右上にオフセット、3枚目を中央下にずらすなど。
- リズミカルで洗練された「崩し」を意識。

### スクロール演出
- 既存サイトの GSAP + ScrollTrigger を活用。
- カードやテキストの**スタガードフェードイン**、パララックス効果。
- ただし演出は控えめに。動きすぎは安っぽくなる。

---

## 4. 視覚的メタファーと動線設計

### モチーフ
- STAR UP のコアメッセージ「AI で統合し、データドリブンな意思決定を実現」を抽象的なビジュアルで表現。
  - 例: 薄い地球儀のワイヤーフレーム、ニューラルネットワーク的なノードライン、データフローを示す流線。
- 具体的すぎるイラスト（握手、ガッツポーズ）は使わない。

### 視線誘導
- ユーザーの目を自然に誘導する「レール」を設計する。
  - 装飾的な斜線・弧線を戦略的に配置。
  - セクション番号（`01` `02` `03`）を大きく配置し、視線のアンカーにする。
  - CTA（「詳細を見る」）への動線を意識した配置。

## カラーパレット

既存サイトのモノトーンベースを継続。アクセントは既存の 3D シーンカラーを踏襲。

| 用途 | カラー | 備考 |
|---|---|---|
| 背景（メイン） | `#ffffff` / `#fafafa` | 白ベース |
| 背景（ダーク） | `#0a0015` / `#000000` | ヒーロー・アクセントセクション |
| テキスト（メイン） | `#000000` | |
| テキスト（セカンダリ） | `#6b7280` (gray-500) | |
| テキスト（ダーク背景） | `#ffffff` / `#d1d5db` (gray-300) | |
| アクセント 1 | `#9b59b6` | 紫 ── 既存ポータルの主色 |
| アクセント 2 | `#3498db` | 青 ── テック感 |
| アクセント 3 | `#e74c3c` | 赤 ── 差し色（控えめに） |

## レスポンシブ戦略

| ブレークポイント | 方針 |
|---|---|
| `lg` (1024px+) | フルのブロークングリッド・オーバーラップ表現 |
| `md` (768px+) | グリッドを簡略化しつつ非対称性は維持 |
| `sm` (〜767px) | シングルカラム。ジャンプ率と余白で雑誌感をキープ |

## 技術スタック（既存踏襲）

- **Next.js** (App Router)
- **Tailwind CSS** (v4)
- **GSAP + ScrollTrigger** (スクロールアニメーション)
- **Three.js** (3D 背景、必要に応じて)
- **Lottie / Spline** (リッチ動的素材、新規導入候補)
