# Recruit（採用情報）Notion DB スキーマ仕様・運用ガイド

採用ページ（`/recruit`）は Notion DB `STAR UP hp-recruit-contents` をCMSとして表示しています。
このドキュメントは **Notion側で設定すべきプロパティの仕様** と、**募集の公開/非公開の運用方法** をまとめたものです。

> ⚠️ **プロパティ名はコードが文字列で参照しています。** 名前を変えると表示が壊れるため、
> 下表の「プロパティ名」は **完全一致** で設定してください（大文字小文字・スペースも含む）。

---

## 1. スキーマ設定（✅ API実施済み）

採用ページのコード改修（複数選択対応・公開ステータス対応）と、**Notion側のスキーマ設定はAPI経由で適用済み**です。
下記が現在のスキーマです。**各行の値（どの職種か・公開するか）は未設定なので、§2 の残作業をお願いします。**

| プロパティ | 型 | 選択肢 | 状態 |
|---|---|---|---|
| `PublishStatus` | セレクト | `募集中` / `募集終了` | ✅ 新規追加済み |
| `Category` | マルチセレクト | `営業` / `マーケティング` / `エンジニア` / `コーポレート` / `人事` / `PM` | ✅ 単一→複数に変換済み |
| `JobType` | マルチセレクト | `法人営業` / `事業開発` / `マーケティング` / `セキュリティエンジニア` / `バックエンドエンジニア` / `フロントエンドエンジニア` / `フルスタックエンジニア` / `インフラエンジニア` / `経理` / `事務` / `人事` / `プロジェクトマネージャー` / `プロダクトマネージャー` | ✅ 単一→複数に変換済み |
| `EmploymentType` | マルチセレクト | `正社員` / `業務委託` / `インターン` | ✅ `インターン` 追加済み |

> ⚠️ 選択肢を「新リストのみ」に置き換えたため、旧選択肢（Categoryの`マネジメント`/`セールス`、JobTypeの`エンジニア`/`マネジメント`/`セールス`）を使っていた行は **その値がクリア**されています（`エンジニア`カテゴリと`プロジェクトマネージャー`職種は新リストにも在るため保持）。§2 で振り直してください。

## 2. 残作業（梅田さん：各行の値を振り分け）

スキーマは整いましたが、**各求人行の値が未設定/クリア済み**です。以下を行うと採用ページが完成します。

1. **`PublishStatus` を全行に設定** — サイトに出す求人は `募集中`、出さない下書き・仕分け行（例: `⬇︎以下新規ページ`）は `募集終了` か空のまま。
   - ※ 1件でも `募集中` を付けると、未設定の行は自動的に非表示になります（§3参照）。今出したい求人には忘れず `募集中` を。
2. **`Category` / `JobType` を各行に設定** — 新しい選択肢から該当するものを（複数可）。
3. **`EmploymentType` を確認** — 既存値は保持済み。インターン求人に `インターン` を付与。

> API適用直後の状態: `エンジニア`(Category)が付いていた5行と`プロジェクトマネージャー`(JobType)の1行のみ値が残り、それ以外のCategory/JobTypeは空です。

## 3. 公開ステータスの挙動（重要）

サイト側は `PublishStatus` を見て表示/非表示を判定します。挙動は次の通りです。

| 状態 | サイト表示 |
|---|---|
| DB内の **誰も** `PublishStatus` を設定していない（＝移行前 / 全部空） | **全件表示**（移行が済むまでの安全フォールバック） |
| DB内で **1件でも** `PublishStatus` を設定した後 | **`募集中` のものだけ表示**。`募集終了` と **未設定（空）** は非表示 |

### ✅ 推奨運用

1. `PublishStatus` を追加したら、**今サイトに出したい求人すべてに `募集中` を付ける**。
2. 作り置きの下書きや、仕分け用の行（例: `⬇︎以下新規ページ` のような区切り行）は **空のまま** にするか `募集終了` にする → サイトに出ません。
3. 募集を止めたいときは `募集終了` に変更（行は消さなくてOK）。再開時は `募集中` に戻すだけ。

> 💡 **`PublishStatus` を1件でも設定すると、未設定の行はすべて非表示になります。**
> 現在サイトに出ている求人は、最初の設定時に忘れず `募集中` を付けてください。

## 4. コードが参照しているプロパティ一覧（リファレンス）

採用ページが読み取っている全プロパティです。名前は完全一致で運用してください。

| プロパティ名 | 型 | 用途 |
|---|---|---|
| `Name` | タイトル | 求人タイトル |
| `ID` | ID（unique_id） | URL（`/recruit/{ID}`）・突合キー |
| `Date` | 日付 | 並び順（降順） |
| `PublishStatus` | セレクト | 公開/非公開（`募集中`/`募集終了`） |
| `Category` | マルチセレクト | カテゴリ（大分類） |
| `JobType` | マルチセレクト | 職種（詳細） |
| `EmploymentType` | マルチセレクト | 雇用形態 |
| `Summary` | テキスト | 概要文 |
| `Location` | テキスト | 勤務地 |
| `Salary` | テキスト | 給与 |
| `WorkingHours` | テキスト | 勤務時間（詳細ページのみ） |
| `Holidays` | テキスト | 休日・休暇（詳細ページのみ） |
| `Benefits` | テキスト | 福利厚生（詳細ページのみ） |
| `Thumbnail` | ファイル | サムネイル画像（未設定時はデフォルト画像） |
| 本文 | ページ本文ブロック | 詳細ページの本文 |

### Category と JobType の使い分け

- **Category（大分類）**: 採用領域のざっくりした分類。フィルタや見出しに使用。
- **JobType（職種詳細）**: 具体的な職種。一覧の絞り込みフィルタ・タグ表示・関連求人の突合に使用。

両者は重複ではなく **「大分類 → 職種詳細」の階層** として運用します。
